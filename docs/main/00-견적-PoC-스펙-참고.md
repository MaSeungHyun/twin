## 1. 서울역사 디지털 트윈 — 견적 산출 관련

### 1.1 목적

서울역사 **대상 공간 3D 모델링** 후, **현재 설치된 서비스와 연동**하여  
**3D 화면 안에서 모니터링**할 수 있게 제작.

### 1.2 3D 모델링 대상 공간


| 층      | 범위                                           |
| ------ | -------------------------------------------- |
| **3층** | 서울역사 3층 전체 (**한화 백화점 제외**)                   |
| **2층** | 대합실 및 환승통로 (**한화 백화점 제외**), **남자 화장실 1소** 포함 |
| **1층** | **플랫폼 전체**                                   |


### 1.3 설치된 서비스 (연동 대상)

#### ① 혼잡도 관리 및 안전관리 감시 (NVR 영상 출력)


| 위치                | 카메라             | 비고                           |
| ----------------- | --------------- | ---------------------------- |
| 2층 환승통로           | **11대** (설치 완료) | 향후 **추가 없음**                 |
| 1층 5·6번 플랫폼 계단 사이 | **10대** (설치 완료) | 추후 전 플랫폼 확대, **약 50~80대** 예정 |


#### ② 화장실 재실 감시


| 위치        | 내용                      |
| --------- | ----------------------- |
| 2층 남자 화장실 | 재실 여부, **위급 사항 이벤트** 도출 |
|           | **추가 설치 없음**            |


※ 추후 추가 서비스 연동 가능 → **별도 견적 협의**

### 1.4 기타 (견적·인프라)

- **On-Premise** 제공 필수 (**납품·프로젝트** — 클라우드 SaaS 불가)
- **우리 PoC**는 별도 — Grapicar 웹 [train-twin.grapicar.com](https://train-twin.grapicar.com) 호스팅 (§6 표)
- 추후 **타 KTX 역사**로 디지털 트윈 확대
- 본 견적을 기반으로 **타 역사 견적** 산출 예정
- **역마다 서비스 구성이 다를 수 있음**
- **서울–부산 간 KTX 역사** 확대 검토 중

## 2. 3D 관제 프로그램 PoC 개요 (1/3)

### 2.1 PoC 성격

- **기능 검증**을 위해 서울역사 **일부 공간**만 대상으로 제작
- 이해 돕기 위한 전달용 (완성 제품 아님)
- **PoC 대상 공간:** 환승통로 / 2층 남자화장실 / **5·6번 플랫폼 일부**

### 2.2 관제 전개 방식

```text
서울역 전체 (이벤트 표시)
    → 세부 영역 (데이터 상세 표시)
```

### 2.3 참조용 디지털 트윈 사이트 (타사 PoC)


| 항목  | 내용                                                                                 |
| --- | ---------------------------------------------------------------------------------- |
| URL | [https://korail.test.bllic.ai/monitoring](https://korail.test.bllic.ai/monitoring) |
| 계정  | admin / korail1234                                                                 |


**우리 입장:** UX·화면 구성 **재기획 필요** (스펙에도 명시). 참조는 **기능·플로우 참고용**이지 그대로 복제 대상이 아님.

### 2.4 통계·저장

- 영역별 **통계 데이터 시각화** (개요 3/3 페이지 참조)
- 저장 주기·데이터는 추후 논의
- **기간 선택** 후 **CSV 또는 EXCEL** 저장 필요

### 2.5 기타 (스펙 원문)

- PoC는 **기능 검증 중심** → **전체 화면·UX 재검토·재기획** 필요
- **추가 제안 가능** (기능, 검증 방식 등)

견적/PoC 개요에는 **열차·안전선**이 문장으로만 암시되거나 빠져 있으나,  
**철도 역사 디지털 트윈** PoC에서는 아래를 **우리 PoC에 명시 포함**한다.


| 추가 항목          | 내용                       | 우선순위              |

| **열차 전광판**     | TAGO 실데이터 스케줄 (오픈 API)   | P0 `F-04`         |

## 7. 프론트 설계 시 체크리스트 (이 문서 기준)

- [ ] 전체 뷰에서 **이벤트**가 보이는가
- [ ] 클릭/알람 시 **세부 영역 + CCTV**로 이어지는가
- [ ] **혼잡/위험** 표시 방식을 1가지 이상 시연할 수 있는가
- [ ] **21대 카메라**를 기능/구역별로 구분해 보여 줄 수 있는가
- [ ] **화장실 재실·위급** 시나리오가 있는가
- [ ] **열차 + 안전선** 시나리오가 있는가 (우리 추가)
- [ ] **통계·CSV**는 P1로 로드맵 제시 가능한가
- [ ] 타사 참조 사이트 대비 **UX가 나은지** 설명 가능한가
