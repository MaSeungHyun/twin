## 1. 프로젝트 한 줄 정의

**서울역사 3D 디지털 트윈 관제 웹 앱** — 역사 공간을 3D로 보여 주고, IoT·CCTV·열차·알람을 **한 화면에서 감시**하며, 이상 발생 시 **해당 구역을 자동 표시**하고 알람 **클릭 시 3D fly-to·CCTV 연동**하는 **PoC 웹 앱**. (프로젝트 납품은 **온프레미스** — §1.1.1)

> **「포커싱」 정의 (F-07, `03-UX` §5.2):**  
>
> - **자동:** 알람 패널·3D 마커·미확인 카운트 — “어디서” 이상인지 즉시 표시  
> - **클릭 후:** 카메라 fly-to, 안전선/재실 강조, CCTV 1채널

### 1.1 제안 맥락 (중요)


| 항목            | 내용                                                                          |
| ------------- | --------------------------------------------------------------------------- |
| **현재 상태**     | **PoC 구현 중** — 발주·계약 전 프로토타입                                                |
| **1차 역사**     | **서울역**                                                                     |

| **프로젝트 인프라**  | **온프레미스** (발주 견적 — 클라우드 SaaS 납품 불가)                                         |

#### 1.1.1 배포 구분 (혼동 방지)


| 구분         | 대상                | 배포                                               | 데이터                           |
| ---------- | ----------------- | ------------------------------------------------ | ----------------------------- |
| **프로젝트**   | LG·코레일 **납품**     | 역사 내 **온프레미스** 서버 (Spring 또는 NestJS, NVR, Redis) | Socket + 실 NVR                |
| **PoC 데모** | **Grapicar 웹 공개** | **Grapicar 홈페이지** 정적 웹 + TAGO **서버 프록시**         | Mock Adapter + TAGO 실 API     |
| **로컬 개발**  | 프론트 구현            | `npm run dev`                                    | Vite `/api/tago` proxy + Mock |


- 견적서 「Cloud 불가」는 **납품 솔루션**에 대한 조건이지, **우리 PoC를 웹에 올리지 말라**는 뜻이 **아님**.
- PoC 빌드(`vite build`)에는 dev proxy가 없음 → **train-twin.grapicar.com** 에서 TAGO는 **서버 1프록시**로 키 보관 (`04` 10절).

즉, 이 프로젝트는 “납품 중인 제품”이 아니라 **기능·UX 검증용 PoC**이다.  
기능 검증용 화면보다 **UI/UX 완성도, 데모 연출, 성능, 확장 구조**가 우선이다.

## 2. 성공 기준 (PoC 데모가 “됐다”는 정의)

데모 당일 아래가 **끊김 없이** 동작하면 PoC 성공으로 본다.

1. **3D 역사 뷰**가 관제 모니터에서 안정적으로 돌아간다 (성능 위젯으로 수치 확인). **위성지도·파노라마 배경 없음.**
2. **전체 뷰 → 구역 진입**이 가능하다 (카메라 전환, Walk/Flyover 버튼 남발 없이 **프리셋·fly-to** 중심).
3. **열차 전광판**에 **실데이터(TAGO)** 스케줄이 표시된다.
4. **열차 3D 연출**이 스케줄 시각과 맞물려 진입·정차하고, **플랫폼 안전선**이 3D에 표시된다.
5. **이벤트 알람** 2종(안전선 침범, 화장실 위급)이 뜨고, 클릭 시 **fly-to → 구역 강조 → CCTV 1채널** (+ 인근 썸네일)가 **한 흐름**으로 이어진다.
6. **화장실 재실** 4칸 상태(공실/사용/위급)가 표시되고, **S3 위급 시나리오**가 동작한다 (F-10).
7. 혼잡·위험 수치가 **Adapter 정규화** 후 **0~100% 또는 4단계 라벨**로 표시된다 (**300%+ UI 금지**).
8. 시연 시 **Mock이지만 프로젝트은 Adapter·Config 교체** 구조를 문서로 설명 가능하다.
9. **Galaxy Tab S9**(가로, Chrome/Samsung Internet)에서 **S1~S3**가 PC와 동일하게 끊김 없이 동작한다 (§8.1).

**3분 데모 스크립트 (권장):** 전체 뷰 → TAGO+열차 → **안전선 침범(S2)** → **화장실 위급(S3)** → 로드맵 한 줄.

### 3.3 PoC에서 **하지 않는 것** (명시적 제외)

구현·데모 준비 시 “왜 없나?”에 바로 답할 수 있도록 적어 둔다.


| 제외 항목                                    | 이유                                      |
| ---------------------------------------- | --------------------------------------- |
| **1~3층 전체 고품질 3D**                       | PoC는 매스/핵심 구역; 전체 GLB는 프로젝트             |
| **NVR 실스트림 21~80대**                      | 연동·인프라·성능 검증은 프로젝트; PoC는 Mock 영상        |

| **한화백화점 내부**                             | 발주 스펙 제외                                |
| **역간 선로·전국 선로 관제**                       | 역사 단위 관제                                |
| **모바일·반응형 최적화**                          | 관제 대형 모니터/PC 전제                         |
| **다국어**                                  | 한국어 관제 UI만                              |
| **Excel보내기**                             | PoC는 **CSV** 우선; Excel은 P1              |
| **타사식 6탭 전부**                            | P0는 실시간 관제만; 확인대기·히스토리·설정은 P2           |

## 4.5 화면·탭 범위 (타사 대비)

PoC는 **“탭 많은 관리자 앱”**이 아니라 **“실시간 3D 관제 한 화면”**을 완성한다.


| 화면/탭            | 타사  | 우리 우선순위         | 비고                                                   |
| --------------- | --- | --------------- | ---------------------------------------------------- |
| **실시간 관제**      | 있음  | **P0 (80% 노력)** | 3D + **플로팅** 알람·전광판·CCTV                             |
| 확인 대기           | 있음  | **P2**          | P0는 알람 패널에서 동일 이벤트 처리                                |
| CCTV (그리드)      | 있음  | **P2**          | P1은 3D 내 카메라 아이콘 + 1채널 상세                            |
| 통계              | 있음  | **P1**          | 차트 + CSV; 3D 연동 하이라이트는 P2                            |
| 히스토리            | 있음  | **P2**          |                                                      |
| 설정 (XYZ·go2rtc) | 있음  | **PoC 화면 제외**   | `devices.json` + 프로젝트 **3D Drag & Drop 노코드 편집** (P2) |


**P0 실시간 관제 레이아웃 (고정) — Floating Overlay:**

```text
┌─────────────────────────────────────────────────────────┐
│ 헤더: 역명 · 시계 · 구역 칩 · 미확인 · Live/Mock 배지      │  ← 고정
├─────────────────────────────────────────────────────────┤
│                                                         │
│              3D 뷰포트 (full-bleed, 크기 고정)            │
│              + 이벤트 마커(최소) + 안전선 mesh            │
│                                                         │
│  ┌알람┐                              ┌전광판/CCTV/재실┐ │
│  │floating│                          │   floating     │ │
│  └────┘                              └────────────────┘ │
│  PerfWidget (좌하)                                      │
└─────────────────────────────────────────────────────────┘
```

평시: 좌·우 패널 **접힘** → 3D **가림 최소**.  
이벤트: 알람 패널 **슬라이드 인** → 클릭 시 fly-to → 우측 **CCTV 1채널** + **동일 구역 인근 2~3 썸네일** · 화장실 알람 시 **재실 패널**.  
**Push 금지:** 패널 열림/닫힘과 **무관하게** 3D canvas 영역은 변하지 않음.

### 8.1 태블릿 (Galaxy Tab S9) — 배경·검증 기준

- **배경:** 외부 이해관계자 측에서 「태블릿에서도 잘 동작하면 좋겠다」는 요구가 있음 (발주사 LG 전용 스펙은 아님).
- **S9를 기준으로 잡는 이유:** 당사 보유 **Galaxy Tab S9** 로 PoC 전·데모 전 **실기 테스트**를 수행하기 위함. 다른 태블릿 완전 호환은 PoC 범위 밖.
- **1차 vs 2차:** 관제실 **PC 대형 모니터**가 주 사용 환경(페르소나·레이아웃·NF-10). 태블릿은 **부가 검증**이지만, S9에서 S1~S3가 통과하지 않으면 PoC 완료로 보지 않음.
- **구현 시 유의 (S9):**
  - **NF-04** DPR 상한(≤1.5) — 고해상도 태블릿에서 GPU 부하 완화
  - **NF-10** floating 패널 — 11" 가로에서 좌·우 패널이 3D를 과도하게 가리지 않도록 평시 **접힘** 우선
  - **터치:** Orbit·패널 스크롤 `pointer-events` 분리 (`03` §11.1) — 태블릿에서 드래그 끊김 없어야 함
  - **CCTV:** 활성 `<video>` 1개 유지 (`F-09`) — 모바일 디코더·autoplay 정책 준수
